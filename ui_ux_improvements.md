# مراجعة UI/UX — الصفحة الرئيسية وموقع «جذع»

> **تاريخ المراجعة:** 9 يونيو 2026  
> **الحالة:** مقترحات للمراجعة — **لم يُنفَّذ شيء بعد**  
> **النطاق:** الصفحة الرئيسية (`index.html`)، الفوتر، شريط التنقل، وتجربة المستخدم العامة

---

## ملخص تنفيذي

الموقع يمتلك هوية بصرية قوية (ألوان خشبية، خط ثمانية، طابع «حكاية تنمو») وبنية SPA تعمل مع Firebase. لكن تجربة المستخدم في الصفحة الرئيسية ما زالت ناقصة في نقاط أساسية: **الفوتر شبه فارغ**، **التنقل غير مكتمل**، **بعض الأقسام المطلوبة غير موجودة**، وهناك **تعارض بين ملفات قديمة ومنفصلة** وبين التطبيق أحادي الصفحة الحالي.

الهدف من هذا الملف: عرض التعديلات المقترحة بوضوح حتى توافق عليها قبل التنفيذ.

## 1. الفوتر — المشكلة الأبرز 🔴

### الوضع الحالي

```html
<footer>
    <img src="assets/logo.png" ...>
    <p>حكاية جذع © 2026. جميع الحقوق محفوظة.</p>
</footer>
```

الفوتر يحتوي فقط على الشعار وحقوق النشر. لا روابط، لا تواصل، لا شبكات اجتماعية.

### ملاحظة مهمة

في لوحة التحكم (`admin/settings.html`) مكتوب: *«معلومات التواصل (تظهر في الفوتر)»* — لكن في الواقع هذه المعلومات تظهر **فقط** في قسم «لنصنع حكاية جديدة» بالصفحة الرئيسية، وليس في الفوتر. هذا تناقض بين الإعدادات والواجهة.

### المقترح: فوتر من 3 أعمدة

```
┌─────────────────────────────────────────────────────────────┐
│  [شعار + شعار نصي]     │  روابط سريعة      │  تواصل معي    │
│  جملة قصيرة عن جذع      │  الرئيسية          │  بريد         │
│                         │  الخدمات           │  هاتف         │
│                         │  المشاريع          │  أيقونات:     │
│                         │  المقالات          │  تليجرام      │
│                         │  تواصل             │  لينكدإن      │
│                         │  اكتب رأيك         │  جيتهب        │
├─────────────────────────────────────────────────────────────┤
│        حكاية جذع © 2026 — جميع الحقوق محفوظة              │
└─────────────────────────────────────────────────────────────┘
```

**تفاصيل التنفيذ المقترحة:**

- تحميل بيانات التواصل من Firestore (`settings/contact`) ديناميكياً في الفوتر (نفس المصدر المستخدم في قسم التواصل).
- روابط التنقل تستخدم `#home`، `#services`، `#projects`، `#articles`، والتمرير لـ `#contact-sec`.
- إضافة رابط **«اكتب رأيك»** → `client/login.html` (بوابة العملاء لكتابة تقييم).
- أيقونات Font Awesome للشبكات الاجتماعية (المكتبة مُحمَّلة أصلاً لكن غير مستخدمة في الروابط الديناميكية).
- على الجوال: الأعمدة الثلاثة تتحول لعمود واحد مكدّس.

## 2. شريط التنقل (Navbar) 🟠

### مشاكل موجودة

| # | المشكلة | التأثير |
|---|---------|---------|
| 1 | **لا يوجد رابط «الخدمات»** في النافبار | المستخدم لا يصل لقائمة الخدمات الكاملة |
| 2 | **لا يوجد مسار `services` في الـ Router** | حتى لو أُضيف الرابط، الصفحة غير معرّفة في `routes` |
| 3 | رابط «تواصل معي» يستخدم `#home` + `onclick` | لا يُحدَّد كـ `active` عند التمرير لقسم التواصل |
| 4 | الشعار غير قابل للنقر | المستخدم يتوقع أن يعيده للرئيسية |
| 5 | **لا قائمة جوال (Hamburger)** | على الشاشات الضيقة الروابط + زر الوضع الليلي تتزاحم |
| 6 | النافبار لا يُخفى عند فتح القائمة | تجربة جوال غير مكتملة |

### المقترح

1. إضافة `<a href="#services">الخدمات</a>` في النافبار.
2. إنشاء دالة `renderServices()` في الـ Router (مشابهة لـ `renderProjects` و `renderArticles`).
3. جعل الشعار رابطاً: `<a href="#home">`.
4. إضافة قائمة جوال:
   - زر ☰ يظهر تحت `600px`.
   - القائمة تنزلق من الأعلى أو الجانب.
   - إغلاق عند النقر على رابط أو خارج القائمة.
5. تحسين `active state`: عند التمرير في الصفحة الرئيسية، تمييز القسم الظاهر (اختياري — أولوية منخفضة).

## 3. أقسام ناقصة في الصفحة الرئيسية 🟠

### 3.1 قسم «أحدث المقالات» — مفقود

حسب `project.md`، الصفحة الرئيسية يجب أن تعرض:
- خدمات مميزة ✅
- مشاريع مميزة ✅
- **أحدث المقالات ❌**

**المقترح:** إضافة قسم بين المشاريع والآراء:

```
أحدث المقالات
[بطاقة 1] [بطاقة 2] [بطاقة 3]
        [عرض جميع المقالات →]
```

- جلب 3 مقالات بترتيب `publishDate` تنازلياً.
- زر «عرض الكل» يوجّه إلى `#articles`.

### 3.2 أزرار «عرض الكل» — مفقودة

أقسام «أبرز الخدمات» و«أبرز المشاريع» تعرض 3 عناصر فقط بدون رابط لرؤية المزيد.

**المقترح:** عنوان القسم يصبح صفاً:

```
أبرز المشاريع                    [عرض الكل ←]
```

### 3.3 زر السيرة الذاتية — غير فعّال

```javascript
onclick="alert('قم برفع ملف السيرة الذاتية أولاً.');"
```

**المقترح:**

1. في `admin/settings.html`: حقل رفع ملف CV إلى Firebase Storage + حفظ الرابط في `settings/general`.
2. في Hero: الزر يحمّل الملف مباشرة أو يفتحه في تبويب جديد.
3. إذا لم يُرفع ملف: إخفاء الزر أو تعطيله مع تلميح بسيط بدلاً من `alert`.

### 3.4 المهارات — ثابتة وليست من قاعدة البيانات

المهارات في Hero مكتوبة يدوياً:

```
JavaScript | HTML5 & CSS3 | Firebase | UI/UX | ...
```

`project.md` يطلب إدارة المهارات من لوحة التحكم — **لا توجد صفحة `admin/skills.html`**.

**المقترح (مرحلتان):**

- **مرحلة 1 (سريعة):** الإبقاء على الوسوم الحالية مع تحسين التنسيق.
- **مرحلة 2:** إنشاء `admin/skills.html` + مجموعة `skills` في Firestore + تحميل ديناميكي في Hero أو قسم «من أنا؟».

### 3.5 رابط «اكتب رأيك» — غير موجود في الواجهة العامة

صفحة `client/write-review.html` موجودة لكن **لا يوجد أي رابط إليها** من الموقع العام.

**المقترح:** إضافتها في الفوتر + رابط ثانوي أسفل قسم الآراء:

> «هل عملنا معاً؟ [شاركنا رأيك]»

## 4. مشاكل UI/UX تفصيلية 🟡

### 4.1 تعارض الأنماط (Inline CSS vs style.css)

`index.html` يحتوي على `<style>` داخلي (~90 سطر) يُعيد تعريف `.hero`، `.reviews-section`، `.contact-section`، `.social-links` — بينما `style.css` يحتوي تعريفات أحدث وأكثر تفصيلاً.

**النتيجة:** تعارض بصري (مثلاً `.social-links a` في الـ inline: نص مسطّر بعرض؛ في CSS: دوائر أيقونات 45×45).

**المقترح:** حذف الـ `<style>` الداخلي من `index.html` والاعتماد على `style.css` فقط.

### 4.2 أيقونات التواصل الاجتماعي

الكود الديناميكي يولّد:

```html
<a href="...">تليجرام</a>  <!-- نص فقط -->
```

بينما CSS يتوقع أيقونات دائرية. **المقترح:** استخدام `<i class="fab fa-telegram">` وما شابه.

### 4.3 آراء العملاء

| نقص | مقترح |
|-----|-------|
| لا نجوم تقييم | إضافة حقل `rating` (1–5) في الإدارة وعرض نجوم في البطاقة |
| التمرير الأفقي غير واضح | تلميح «اسحب للمزيد ←» أو أسهم تنقل |
| لا تاريخ للرأي | عرض تاريخ اختياري |

### 4.4 نموذج التواصل — إمكانية الوصول

- الحقول بدون `<label>` — فقط `placeholder`.
- **المقترح:** إضافة تسميات مرئية أو `aria-label`.
- إضافة رسالة نجاح/خطأ واضحة فوق الزر (وليس تغيير نص الزر فقط).

### 4.5 قسم «من أنا؟»

- لا صورة شخصية أو رسم للشخصية «جذع».
- **المقترح:** صورة/رسم بجانب النص (شبكة 1:2 على الديسكتوب، مكدّسة على الجوال).
- يمكن رفع الصورة من الإعدادات.

### 4.6 متغير CSS غير معرّف

`.error-state` يستخدم `var(--danger)` لكن `--danger` غير موجود في `:root`.

**المقترح:** إضافة `--danger: #c62828;` (أو لون متناسق مع الهوية).

### 4.7 الوضع الليلي

`project.md` الأصلي يقول: *«لا أريد الوضع الليلي»* — لكنه مُضاف حالياً.  
**قرار مطلوب منك:** الإبقاء عليه أم إزالته؟ (المراجعة لا تفترض إزالته إلا إذا طلبت ذلك.)

## 5. ديون تقنية وتناقض معمارية 🔵

### 5.1 صفحات HTML منفصلة قديمة

الملفات التالية ما زالت موجودة وتعمل بشكل مستقل:

- `projects.html`
- `articles.html`
- `services.html`
- `project.html`
- `article.html`

بينما `index.html` أصبح SPA كاملاً بنفس الوظائف عبر Hash Router.

**المشاكل:**

| المشكلة | مثال |
|---------|------|
| ترويسة مختلفة | الصفحات القديمة: شعار نصي فقط بدون صورة |
| لا وضع ليلي | الصفحات القديمة تفتقر للميزات |
| لا فوتر غني | نفس الفوتر الفارغ |
| SEO مزدوج | محركات البحث قد تفهرس نسختين |
| صيانة مضاعفة | أي تعديل يحتاج تكراراً |

**المقترح:**

- **الخيار أ (مُوصى به):** حذف الصفحات المنفصلة أو تحويلها لإعادة توجيه (`index.html#projects`).
- **الخيار ب:** الإبقاء عليها مع مزامنة التصميم (أكثر جهداً).

### 5.2 JavaScript مدمج في HTML

كل منطق الـ SPA (~400 سطر) داخل `index.html`.

**المقترح:** نقله إلى `js/app.js` و`js/router.js` حسب هيكل `project.md` — يسهّل الصيانة دون تغيير السلوك.

### 5.3 SEO ومعاينة الروابط

- لا `meta description`.
- لا Open Graph (`og:title`, `og:image`, `og:description`).
- عناوين الصفحات الفرعية لا تتغير ديناميكياً (`document.title`).

**المقترح (أولوية متوسطة):**

```javascript
// عند فتح مقال
document.title = `${data.title} - جذع`;
// تحديث meta description ديناميكياً
```

## 9. قرارات تحتاج موافقتك

قبل التنفيذ، أحتاج تأكيدك على:

1. **الفوتر:** هل التصميم المقترح (3 أعمدة) مناسب؟ أم تفضل فوتراً أبسط (روابط + حقوق نشر فقط)؟
2. **الوضع الليلي:** إبقاء أم إزالة؟
3. **الصفحات القديمة:** حذف وإعادة توجيه أم مزامنتها؟
4. **المهارات:** مرحلة 1 (ثابتة محسّنة) أم مرحلة 2 (من لوحة التحكم) مباشرة؟
5. **السيرة الذاتية:** رفع PDF من الإعدادات — هل هذا مقبول؟
6. **أي أقسام إضافية** تريدها في الرئيسية؟ (مثلاً: إحصائيات، شريط شعارات عملاء، جدول أسعار)
